Game

Let's create a simple game using Vuforia. Vuforia uses the device's camera feed combined with accelerometer and gyroscope data to examine the world. So, this tutorial creates a simple game where spheres appears randomly in different directions and speed. This allows the user to follow the spheres around with the device. It also add a tap interaction. As soon as the user tap on the sphere, it disappears.


Initial Setup

Vuforia is one of the most solid augmented reality tools. It is an AR library that is compatible with Android, iOS, UWP, and some brands of smart glasses.


Vuforia Configuration

Before we start playing around, we’ll need to prepare Unity for Vuforia. It is quite simple process because since the 2017.2 version, Unity integrates the Vuforia Engine. You just have to make sure to select Vuforia Augmented Reality Support in the Components section when installing Unity. Also, don't forget to install your platform support (iOS, Android, UWP). This tutorial was done by using Android.

First, we need to activate Vuforia Engine in the project.

  • Go to Player Settings to activate Vuforia Engine under the "XR Settings" section and check "Vuforia Augmented Reality"
  • Delete "Main Camera"
  • Add "AR Camera" by going to GameObject > Vuforia > AR Camera.
  • Create/Log in a developer account on Vuforia.
  • Go to License Manager on Vuforia’s developer portal and get a new development key
  • Put the name of your project as the License name and create the license.
  • After creating it, select the license from the list (if you have more) and copy the license key
  • Back to Unity, select ARCamera element. In the Inspector panel, click on the button "Open Vuforia Engine configuration" below the Vuforia Behavior (Script) component
  • Find the field "App License Key", and paste the license you created in Vuforia's developer portal.
  • Test the configuration by building your project on your device. For now, all that you should expect to see is the camera feed on your device without any error
    • Android:Vuforia Engine does not support Android TV. Disable the Android TV Compatibility in the Player Settings > Other Settings > Configuration

Now you can add any features of Vuforia Engine to your project.

Game

For this specific game, there is still missing one Vuforia Configuration. Vuforia needs to track the position of the device relative to the world.

  • Open Vuforia Engine configuration and activate Track Device Pose

Let's spawn ours spheres.
  • Create a Empty Object and add a new C# script. You can delete the Update function. We will not used it.
  • Let's create a function to set the initial position of this empty object.
    • We will get the camera position and set this empty object to be spawned 10 units forward from the camera.
    • We'll set the position only one time within the Start function.
    • Let's create a coroutine to wait a little time to set the position.
// if position was set
private bool isPositionSet;
// Start is called before the first frame update
void Start()
{
    StartCoroutine(ChangePosition());
}

private bool SetPosition()
{
    // get the camera position
    Transform cam = Camera.main.transform;

    // set the position 10 units forward from the camera position
    transform.position = cam.forward * 10;
    Debug.Log("Setting position to:" + transform.position);
    return true;
}

private IEnumerator ChangePosition()
{

    yield return new WaitForSeconds(0.2f);
    // Define the Spawn position only once
    if (!isPositionSet)
    {
        // change the position only if Vuforia is active
        if (VuforiaBehaviour.Instance.enabled)
            isPositionSet = SetPosition();
    }
}
  • We already have set the initial position of our empty object based on the device camera postion.
  • Now we'll create an area around our empty object to establish where our spheres will spawn.
  • Next step, we'll change the script to instantiate the spheres with different speed and directions.
  • Create a coroutine to spawn the elements.
// reference to the gameObject that will be instantiate
public GameObject enemy;

// Time to spawn the enemies
public float timeToSpawn = 0.5f;

private IEnumerator SpawnLoop()
{
    // Defining the Spawning Position
    StartCoroutine(ChangePosition());

    yield return new WaitForSeconds(0.2f);

    InvokeRepeating("Spawn", timeToSpawn, timeToSpawn * 2);
}

void Spawn()
{
    // Create an instance of the enemy prefab at the randomly selected spawn point's position and rotation.
    transform.position = transform.position + Camera.main.transform.forward * Time.deltaTime;
    Instantiate(enemy, (Random.insideUnitSphere * 4) + transform.position, transform.rotation);
}
  • Edit the Start function to call the spawn coroutine instead of the ChangePosition coroutine.
// Start is called before the first frame update
void Start()
{
    StartCoroutine(SpawnLoop());
}
  • Create a sphere prefab to be instantiated by the script
  • Drag the prefab to the enemy field of our script


Let's add movement to ours spheres.
  • Add a new C# script to the sphere prefab
  • Each sphere will have random characteristics. Rotation speed and rotation direction will be defined randomly.
// Orbit max Speed
public float orbitMaxSpeed = 30f;

// Orbit speed
private float orbitSpeed;

// Anchor point for the element to rotate around
private Transform orbitAnchor;

// Orbit direction
private Vector3 orbitDirection;

// Growing Speed
public float growingSpeed = 10f;

void Start()
{
    ElementSettings();
}

// Set initial element settings
private void ElementSettings()
{
    // defining the anchor point as the main camera
    orbitAnchor = Camera.main.transform;

    // defining the orbit direction
    float x = Random.Range(-1f, 1f);
    float y = Random.Range(-1f, 1f);
    float z = Random.Range(-1f, 1f);
    orbitDirection = new Vector3(x, y, z);

    // defining speed
    orbitSpeed = Random.Range(5f, orbitMaxSpeed);
}
  • Add moviment to the sphere
    // Update is called once per frame
    void Update()
    {
        // makes the element orbit and rotate
        RotateElement();
    }

    // Makes the element rotate around a anchor point
    // and rotate around its own axis
    private void RotateElement()
    {
        // rotate element around camera
        transform.RotateAround(
            orbitAnchor.position, orbitDirection, orbitSpeed * Time.deltaTime);

        // rotating around its axis
        transform.Rotate(orbitDirection * 30 * Time.deltaTime);
    }

Conclusion

This is just the basics for a AR game. You could add particle explosion, create a score system, adding a timer to improve the game. Vuforia provides sophisticated resources that can create engaging experiences. They are Model Targets, Image Targets, Object Targets, Multi-Targets, Cylinder Targets, VuMarks, External Camera and Ground Plane.

using System.Collections;
using System.Collections.Generic;
using UnityEngine;

public class ElementBehavior : MonoBehaviour
{

    // Orbit direction

    void Start()
    {
        ElementSettings();
    }

    // Set initial element settings

    {
